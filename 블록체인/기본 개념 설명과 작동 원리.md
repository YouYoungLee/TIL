# Digital Signature
- 디지털 서명

# Blockchain Network
## 블록체인 네트워크
## 중앙집중형 네트워크
- 모든 데이터와 정보가 중앙 서버에 저장되어 있음
- 서버가 다운되면 네트워크 전체에 영향을 미침
- 대표적으로 은행, 인터넷 쇼핑몰 등이 해당됨
## 분산형 네트워크
- 네트워크에 참여하는 모든 노드가 데이터와 정보를 동시에 가지고 있음
- 한 노드가 다운되어도 네트워크 전체에 영향을 미치지 않음
- 대표적으로 비트코인, 이더리움 등이 해당됨
## Blockchain Network, How to Work
- 블록체인 네트워크 작동방식
- 블록체인 네트워크에서는 블록이라는 단위로 데이터를 저장함
- 블록은 이전 블록에 대한 정보와 현재 블록에 대한 정보를 가지고 있음
- 이전 블록에 대한 정보를 가지고 있기 때문에 블록이 변조되면 이전 블록의 정보와 맞지 않아서 검증이 불가능해짐
## Data in Block
- 블록 안 데이터
- 블록의 해쉬는 헤더를 가지고 있고, 이전 해시, 임시값, 루트해시에 대한 데이터를 가지고 있다
- 블록의 데이터는 거래 정보, 즉 송금자와 수취인의 주소, 송금량, 타임스탬프, 그리고 다양한 메타데이터 등이 포함됨

# 정리
1. 블록체인은 보안성, 투명성, 무결성, 탈중앙성의 특징을 가지고 있고, 이 특징으로 여러가지 어플리케이션을 활용할 수 있다.
2. 블록체인의 네트워크는 기본적으로 P2P 네트워크에서 작동한다.
3. 암호화에는 복호화 가능 암호화, 단방향 암호화로 나눠지고, 단방향 암호화는 해시함수로 SHA256 keccak256등이 있다
4. 암호화에는 서명과 확인을 이용해 메시지의 무결성과 송신자를 보증하는 알고리즘을 전자서명이라고 한다.
5. 이 ECC 암호화 알고리즘은 공개키 암호화로 타원 곡선 함수를 이용하여 암호화 한다. + 디지털서명 = ECDSA

- 메인넷이란 쉽게 설명하자면 기존에 다른 플랫폼을 활용해서 구현된 토큰이 자체 독립된 플랫폼을 구축하고 새롭게 생태계를 구성하는 것을 뜻합니다.

# 가스란?
- 이더리움 가스는 트랜잭션을 마치거나 스마트 컨트랙트를 실행하기 위한 계산 비용을 측정하는 단위 이더리움 네트워크에서 수행하는 모든 것은 가스 비용이 발생하며, 이 가스는 ETH로 전환됩니다. 
- 필요한 가스 양은 두 가지 주요 요소에 의해 달라집니다
1. 컨트랙트의 복잡한 정도
2. 완료되는 속도
- 만약 트랜잭션이 완료되는 것을 조금 더 기다릴 의향이 있다면, 더 적은 가스를 지불할 수 있습니다.
- 사용하는 이유
1. 네트워크 과부하 방지 및 트래픽 관리
2. 가스 가격, 가스 한도 설정으로 채굴자들의 우선 순위 최적화
3. 트랜잭션이 너무 비싸지지 않도록 하는 안전 장치 역할
- 알아야 할 용어 
1. Gwei
- Gwei는 이더의 작은 단위로, 각 가스 단위에 부착되어 있습니다. 이는 채굴자들에게 금액적인 인센티브로 환산되어 나타납니다.
2. 가스 한도
- 가스 수수료의 과도한 지불을 방지하기 위하여 만들어진 gas limit(가스한도)입니다. 내가 설정한 기위 값에 가스 한도를 곱해서 수수료를 계산하게 되고 그 한도 내에서 전송을 하거나 어떠한 행동을 취할 수 있게 함 설정한 한도가 필요한 수준을 초과하면 작업이 제 시간에 완료되고 나머지 ETH가 반환됩니다.

# 트랜잭션이란 
- 가상자산을 참여자 간 전송, 수신할 때 저장되는 거래 기록입니다. 공통적으로 개별 검증자들이 트랜잭션의 내용을 검증하게 되고, 과반수의 검증자가 그 내용에 동의해야 합니다.고유한 ID(트랜잭션 해시값)를 가지며, 이를 이용해 암호 화폐의 전달 여부, 그리고 거래 정보의 유호성을 검사합니다. UTXO "소비되지 않은 거래 출려값" 거스름돈 느낌 
1. Txn Hash
- 트랜잭션 해시값, 즉 해당 트랜잭션의 고유 ID입니다
2. Method
- 해당 트랜잭션이 수행한 작업을 설명합니다. 예를 들어 토큰 전송은 Transfer,NFT 민팅은 Mint로 표시됩니다.
3. Block
- 해당 트랜잭션이 발생한 블록 높이(거래 시간)입니다. 위 예시에는 15375516 블록의 트랜잭션이 표시됩니다.
4. Age/Date Time(UTC)
- 트랜잭션이 발생한 시간을 표시합니다. 최소 표시 형식은 "Age"로, 조회 시점을 기준 "Age" 텍스트 클릭 시 날짜-시간 형식으로 바뀝니다
5. From
- 보내는 지갑의 주소입니다. 클릭 시 해당 지갑의 트랜잭션 목록이 표시됩니다.
6. To
- 받는 지갑의 주소입니다. 클릭 시 해당 지갑의 트랜잭션 목록이 표시됩니다
7. Txn Free
- 해당 트랜잭션이 지불한 트랜잭션 수수료 금액입니다.
