# 배워야 하는 이유
1. HTML 문서의 콘텐츠를 동적으로 변경할 수 있는 언어
2. Web 이라는 공간에서 채팅, 게임 등 다양한 동작을 할 수 있게 된 기반
3. 단순히 Web 조작을 넘어서 서버 프로그래밍, 모아빌 서비스, 컴퓨터 응용프로그래밍, 블록체인,게임 개발 등 다양한 분야에서 활용이 가능한 언어가 됨
## 웹 브라우저의 역할
- HTML/CSS/JavaScript를 이해한 뒤 해석해서 사용자에게 하나의 화면으로 보여줌
- 즉, 웹 페이지 코드를 이해하고, 보여주는 역할을 하는 것이 바로 웹 브라우저
# 코드 작성법
## 세미콜론(semicolon)
- 자바스크립트는 세미콜론을 선택적으로 사용 가능
- 세미콜론이 없으면 ASI에 의해 자동으로 세미콜론이 삽입됨(Automatic Semicolon Insertion, 자동 세미콜론 삽입 규칙)
## 들여쓰기와 코드 블럭
- python은 4칼 들여쓰기를 사용했으나, JavaScript는 2칸 들여쓰기를 사용
- 블럭(block)은 if,for,함수에서 중괄호 {} 내부를 말함
1. python은 들여쓰기를 이용해서 코드 블럭을 구분
2. JavaScript는 중괄호 {}를 사용해 코드 블럭을 구분
## 주석
- 한 줄 주석(//)과 여러 줄(/**/)주석
# 변수와 식별자
## 식별자 정의와 특징
1. 식별자(identifier)는 변수를 구분할 수 있는 변수명을 말함
2. 식별자는 반드시 문자, 달러($) 또는 밑줄(_)로 시작
3. 대소문자를 구분하며, 클래스명 외에는 모두 소문자로 시작
4. 예약어 사용 불가능(for,if,function 등)
## 변수 선언 키워드
1. let
- 블록 스코프 지역 변수를 선언 (추가로 동시에 값을 초기화)
2. const
- 블록 스코프 읽기 전용 상수를 선언 (추가로 동시에 값을 초기화)
3. var
- 변수를 선언 (추가로 동시에 값을 초기화)
## let
1. 재할당 가능 & 재선언 불가능
2. 블록 스코프를 갖는 지역 변수를 선언, 선언과 동시에 원하는 값으로 초기화 할 수 있음
## const
1. 재할당 불가능 & 재선언 불가능
2. 선언 시 반드시 초기값을 설정 해야 하며, 이후 값 변경이 불가능
3. let과 동일하게 블록 스코프를 가짐
## var
1. 재할당 가능 & 재선언 가능
2. "호이스팅" 되는 특성으로 인해 예기치 못한 문제 발생 가능
3. 따라서 ES6 이후붜는 var 대신 const와 let을 사용하는 것을 권장
4. 함수 스코프(function scope)를 가짐
## 호이스팅
1. 변수를 선언 이전에 참조할 수 있는 현상
2. var로 선언된 변수는 선언 이전에 참조할 수 있으며, 이러한 현상을 호이스팅이라 함
3. 변수 선언 이전의 위치에서 접근 시 undefined를 반환
## 기본적으로 const 사용을 권장 재할당해야 하는 경우만 let
## Number
1. NaN
- Not-A-Number(숫자가 아님)를 나타냄
- Number.isNaN() 의 경우 주어진 값의 유형이 Number이고 값이 NaN이면 true, 아니면 false를 반환
## NaN을 반환하는 경우
1. 숫자로서 읽을 수 없음
2. 결과가 허수인 수학 계산식
3. 피연산자가 NaN
4. 정의할 수 없는 계산식
5. 문자열을 포함하면서 덧셈이 아닌 계산식
## String
- 문자열을 표현하는 자료형
- 작은 따옴표 또는 큰 따옴표 모두 가능
- 곱셈, 나눗셈, 뺄셈은 안되지만 덧셈을 통해 문자열 붙일 수 있음
- Quote를 사용하면 선언 시 줄 바꿈이 안됨
- 대신 escape sequence를 사용할 수 있기 때문에 \n를 사용해야 함
- Template Literal을 사용하면 줄바꿈이 되며, 문자열 사이에 변수도 삽입도 가능
## Template literals(템플릿 리터럴)
1. 내장된 표현식을 허용하는 문자열 작성 방식
2. ES6+ 부터 지원
3. Backtick('')을 이용하며, 여러 줄에 걸쳐 문자열을 정의할 수도 있고 JavaScript의 변수를 문자열 안에 바로 연결할 수 있는 이점이 생김
4. 표현식을 넣을 수 있는데, 이는 $와 중괄호( $ {expression})로 표기
## Empty Value
1. 값이 존재하지 않음을 표현하는 값으로 JavaSCRIPT에서는 null과 undefinde가 존재
2. 동일한 역할을 하는 이 두개의 키워드가 존재하는 이유는 단순한 JavaScript의 설계 실수
3. 큰 차이를 두지 말고 interchangeable 하게 사용할 수 있도록 권장함
## null
- null 값을 나타내는 특별한 키워드
- 변수의 값이 없음을 의도적으로 표현할 때 사용하는 데이터 타입
## undefinde
- 값이 정의되어 있지 않음을 표현하는 값
- 변수 선언 이후 직접 값을 할당하지 않으면 자동으로 할당됨
## null 과 undefinde
1. null과 undefined의 가장 대표적인 차이점은 typeof 연산자를 통해 타입을 확인 했을 때 나타남
2. null 이 원시 타입임에도 불구하고 object로 출력되는 이유는 JavaScript 설계 당시의 버그를 지금까지 해결하지 못한 것
3. 쉽게 해결 할 수 없는 이유는 이미 null 타입에 의존성을 띄고 있는 많은 프로그램들이 망가질 수 있기 때문(하위 호환 유지)
## Boolean
1. true와 false
2. 참과 거짓을 표현하는 값
3. 조건문 또는 반복문에서 유용하게 사용
- 조건문 또는 반복문에서 boolean이 아닌 데이터 타입은 자동 형변환 규칙에 따라 true 또는 false로 변환됨
## 연산자
1. 할당 연산자(++,+=,-=)
2. 비교연산자(<,>)
3. 동등 연산자(==)
4. 일치 연산자(===)
- 두 비교 대상이 타입과 값 모두 같은 지 비교하는 방식
5. 논리 연산자(&&,||,!)
6. 삼항 연산자
- 3개의 피연산자를 사용하여 조건에 다라 값을 반환하는 연산자
## 조건문의 종류와 특징
1. if statement
- 조건 표현식의 결과값을 boolean 타입으로 변환 후 참/거짓을 판단
2. switch statement
- 조건 표현식의 결과값이 어느 값(case)에 해당하는지 판별
- 주로 특정 변수의 값에 따라 조건을 분기할 때 활용
- 조건이 많아질 경우 if문보다 가독성이 나을 수 있음
## if statement
- if,else if ,else
1. 조건은 소괄호(condition)안에 작성
2. 실행할 코드는 중괄호{} 안에 작성
3. 블록 스코프 생성
## switch statement
1. 표현식(exxpression)의 결과값을 이용한 조건문
2. 표현식의 결과값과 case문의 오른쪽 값을 비교
3. break 및 default문은 [선택적]으로 사용 가능
4. break문이 없는 경우 break 문을 만나거나 default문을 실행할 때까지 다음 조건문 실행
5. 블록 스코프 생성
# 반복문

- 조건문이 참이기만 하면 문장을 계속해서 수행
2. for
- 특정한 조건이 거짓으로 판별될 때까지 반복
